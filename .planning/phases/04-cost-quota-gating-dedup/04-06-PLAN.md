<objective>
Populate `ImportJob.costCents` at each pipeline stage boundary with raw units + estimated cents (derived from the `env.import` price table), emit an aggregate-only structured cost log, and add the single-point quota refund inside `failJob()` — keyed by the reserved day (`job.insertedAt`), the only code path that ever writes `status: "failed"`.

Purpose: COST-02/D-03 requires per-stage cost telemetry — raw units (download bytes, ASR minutes, LLM tokens in/out, embedding) plus estimated cents recorded on ImportJob.costCents. COST-01/D-07 requires a failed job to refund the reserved quota exactly once; `failJob` is the traced single-safe refund point (worker `TERMINAL_STATUSES` no-op prevents re-entry). D-08 forbids inline price constants and forbids logging payloads.
Output: cost recording at stage boundaries, an aggregate cost log line, `extractImportedRecipe` exposing token usage, and `refundDailyImportQuota` wired into `failJob`.
</objective>
