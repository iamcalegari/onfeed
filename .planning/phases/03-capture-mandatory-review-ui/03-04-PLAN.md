<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /import (via startImportAction) | User-supplied URL crosses to the backend; client validation is UX-only, the server allowlist is the real gate |
| clipboard → PasteLinkButton | Clipboard contents read under user gesture; untrusted text pre-fills the input (still re-validated server-side on submit) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-07 | Tampering (SSRF boundary bypass) | Client-side URL validation | medium | mitigate | Client isLikelyUrl is UX-only; POST /import's server detectPlatform() allowlist stays the sole SSRF boundary — no second divergent client/server validator introduced. |
| T-03-08 | Information Disclosure | Clipboard read error surfacing | low | mitigate | readText() failures fall back silently (Pitfall 1) — no error text that could reveal clipboard/permission state; paste-event fallback always available. |
| T-03-SC | Tampering | npm/pip/cargo installs | high | accept | No new packages introduced by this plan (RESEARCH §Standard Stack: zero new deps); no install task — supply-chain checkpoint N/A. |
</threat_model>
