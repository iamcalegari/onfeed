Build the mandatory review/edit screen (`/import/[jobId]/review`) and the "Minhas importações" list (`/import/mine`) — the last two Phase 3 screens and the heart of the Core Value safety net. Purpose: Deliver REV-01 (visible review, no silent save), REV-02 (grounding badges), REV-03 (inline edit of every field), REV-04 (confirm gate), and D-09 (owner-scoped imports list).

Output: `/import/[jobId]/review` route + `ImportReviewForm`, `/import/mine` route + `ImportsList`.

Data fetch (resolved Open Question 2): two round-trips — the review page does `GET /import/:jobId` (for recipeId + terminal state) then reuses the existing `GET /recipes/:recipeId` for the full Recipe + grounding. The Recipe is NOT embedded in the polling response.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → PATCH /import/:jobId/recipe (via confirmImportRecipeAction) | User-submitted edited recipe fields cross here; the backend (03-01) is the authoritative gate (owner-scope, content-only schema), the client must not send protected fields |
| browser → /import/mine, /import/[jobId]/review | Authenticated pages; the backend routes (03-01/03-02) enforce owner-only data — the client renders only what it's allowed to fetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-09 | Tampering | Confirm payload (client-side) | medium | mitigate | ImportReviewForm builds the patch from editable content fields only (title/intro/ingredients{name,quantity,unit}/steps{text}) — never grounding/reviewRequired/confidenceScore; backend additionalProperties:false is the hard gate (03-01), client cooperates. |
| T-03-10 | Information Disclosure | Review page reading a private import | high | mitigate | The review page fetches GET /recipes/:recipeId, which (03-02) returns a private import only to its owner (importJobId→ImportJob.userId); a non-owner gets 404 — no unreviewed content leaks. |
| T-03-11 | Tampering | Grounding index-drift → wrong badge | low | mitigate | grounding zipped with its ingredient/step ONCE at fetch time; no groundingArray[index] inside render .map() (Pitfall 4). |
| T-03-SC | Tampering | npm/pip/cargo installs | high | accept | No new packages introduced by this plan (RESEARCH §Standard Stack: zero new deps); no install task — supply-chain checkpoint N/A. |
</threat_model>
