<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SQS message → worker | Message body is minimal ({ jobId }); the authoritative state is the re-read ImportJob doc, not the message |
| worker → local temp disk | Raw third-party media lives transiently on disk and must never leak (retention + cleanup) |
| worker → S3 | Only the derived keyframe crosses to S3; raw media must never be uploaded |
| worker → external platforms | Repeated failed downloads must degrade (breaker), not hammer |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Tampering | SQS message contents driving processing | medium | mitigate | handleMessage trusts only { jobId } and re-reads the authoritative ImportJob doc via findById; terminal-status jobs are no-ops (idempotency) so a replayed/forged message cannot re-run or corrupt a finished job |
| T-05-02 | Information Disclosure | Raw video/audio persisting on disk or leaking to S3 | high | mitigate | Two-layer cleanup (try/finally per job + startup sweep for SIGKILL) guarantees raw media deletion (PIPE-05); keyframe is the ONLY S3 write (imports/{jobId}/keyframe.jpg) — raw media never uploaded (D-09) |
| T-05-03 | Denial of Service | Unbounded concurrent ffmpeg/yt-dlp processes; hammering a blocked platform | high | mitigate | p-queue caps concurrency (2-3); circuit breaker short-circuits blocked platforms and anti_bot_blocked fails explicitly without immediate SQS retry (PIPE-07) |
| T-05-04 | Denial of Service | Duplicate concurrent processing from a too-short visibility timeout | medium | mitigate | visibilityTimeout set to 20 min (~6x p95) per RESEARCH Pitfall 4; idempotency guard is the backstop if redelivery still occurs |
| T-05-05 | Tampering | Temp path collision across concurrent jobs | low | mitigate | mkdtemp with a random suffix (ASVS V12); temp paths derived from server-generated ids, never user input |
| T-05-SC | Tampering | npm installs (sqs-consumer, p-queue) | high | mitigate | Both approved in RESEARCH Package Legitimacy Audit (bbc / sindresorhus orgs, high downloads); sqs-consumer was covered by the blocking-human legitimacy checkpoint in Plan 03 |
</threat_model>
