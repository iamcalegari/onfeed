## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker → ffmpeg binary | Attacker-influenced file paths / URLs must never reach a shell string |
| downloaded media → ffmpeg decode | A hostile/oversized media file could exhaust CPU/memory during decode |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | ffmpeg invocation (ffmpeg.exec.ts, vad.ts, keyframe.ts) | high | mitigate | All three modules invoke ffmpeg via execFile with a discrete args array (never child_process.exec string form) — shell metacharacters in any path are never interpreted (ASVS V12, RESEARCH Security command-injection row) |
| T-02-02 | Denial of Service | ffmpeg decode of a pathological/huge video | high | mitigate | extractAudio downsamples to mono 16kHz 64k (bounds output); the max-duration cap (env.import.maxDurationSec from Plan 01) is enforced upstream before download; p-queue concurrency cap (Plan 05) bounds concurrent decodes |
| T-02-03 | Denial of Service | Worker hammering a blocked platform | high | mitigate | platform-breaker opens after FAILURE_THRESHOLD and short-circuits attempts during COOLDOWN_MS — the graceful-degradation control PIPE-07 requires |
| T-02-SC | Tampering | npm installs | high | mitigate | No new packages installed in this plan (reuses existing sharp + node stdlib child_process); nothing to legitimacy-check |
