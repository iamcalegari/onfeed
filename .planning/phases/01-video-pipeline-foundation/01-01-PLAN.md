<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env/config → runtime | Missing/misconfigured secrets must degrade one feature, not crash the whole worker |
| Mongo document → worker | ImportJob doc is the trust anchor for idempotency; its _id (server-generated) is what the SQS message references |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Denial of Service | Missing GROQ_API_KEY/OPENAI_API_KEY at worker boot | medium | mitigate | Use optional()+enabled (never required()) for transcription keys so a missing key fails one job (transcription_failed) rather than crashing the process — mirrors env.mp.enabled precedent |
| T-01-02 | Tampering | ImportJob _id used as SQS jobId | medium | mitigate | _id is a server-generated Mongo id, never user-supplied; the message body carries only { jobId }, and the worker re-reads the authoritative doc rather than trusting message contents |
| T-01-03 | Denial of Service | Unbounded video duration accepted downstream | high | mitigate | Add env.import.maxDurationSec (default 600s) config here so validation (Plan 04) and the worker (Plan 05) can enforce the cap — bounds the resource-exhaustion blast radius (RESEARCH Security DoS row) |
| T-01-SC | Tampering | npm install (vitest) | high | mitigate | vitest is a mainstream, high-trust devDependency (no [ASSUMED]/[SUS] flag in RESEARCH Package Legitimacy Audit); no other new packages installed in this plan |
</threat_model>
