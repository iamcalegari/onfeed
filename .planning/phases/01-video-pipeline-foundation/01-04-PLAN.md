<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /import | Untrusted URL string crosses into the system; must be validated before any downstream fetch |
| client → GET /import/:jobId | Untrusted jobId; must not expose another user's job |
| validation → yt-dlp (downstream) | The allowlist here is the last line before yt-dlp fetches an arbitrary URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Information Disclosure | SSRF via crafted "video URL" (e.g. http://169.254.169.254/…, internal host) submitted to POST /import | high | mitigate | detectPlatform rejects anything not matching the three public platform domain patterns (and non-http(s) schemes) and returns 400 BEFORE the job is created or the URL reaches yt-dlp — domain allowlisting at validation is the security boundary (RESEARCH Security SSRF row) |
| T-04-02 | Broken Access Control / Information Disclosure | Job-id enumeration on GET /import/:jobId exposing another user's transcript/keyframe/metadata | high | mitigate | Ownership enforced by scoping the query to the caller's userId (getImportJob(jobId, userId)); non-owner receives notFound so existence is not leaked (RESEARCH Security V4 IDOR row) — new attack surface, explicit check required |
| T-04-03 | Denial of Service | Very long/oversized video accepted at submission | medium | mitigate | env.import.maxDurationSec is enforced in the downloader (Plan 03) after metadata; the max-duration constant is defined at config so the boundary is consistent; p-queue concurrency (Plan 05) bounds blast radius |
| T-04-04 | Spoofing | Unauthenticated import submission | medium | mitigate | requireAuth preHandler on both routes reuses the existing Clerk guard unmodified (ASVS V2) |
| T-04-SC | Tampering | npm installs | high | mitigate | No new packages installed in this plan (reuses @aws-sdk/client-sqs, fastify, typebox already in stack) |
</threat_model>
