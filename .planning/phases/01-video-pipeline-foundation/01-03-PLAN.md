<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-submitted URL → yt-dlp | yt-dlp will fetch whatever URL it is handed; the URL crosses from untrusted input into an outbound fetch |
| worker → Groq/OpenAI APIs | Extracted audio (derived from third-party media) is sent to external ASR providers |
| yt-dlp stderr → classifier | Unstructured external error text drives failure routing + circuit-breaker signal |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | URL/args passed to youtube-dl-exec | high | mitigate | youtube-dl-exec takes a typed options object → args array (never a shell string); the URL is never shell-interpreted (RESEARCH Security command-injection row). Domain allowlisting happens upstream at validation (Plan 04) BEFORE the URL reaches this adapter |
| T-03-02 | Denial of Service | Oversized/long video download + ASR | high | mitigate | env.import.maxDurationSec cap enforced after fetchMetadata before download; audio size guard before the Groq call; format "best[ext=mp4]/best" avoids always grabbing the highest-bitrate stream |
| T-03-03 | Information Disclosure | Raw yt-dlp stderr logged | low | accept | stderr is retained on DownloadError for debuggability (A1) and logged worker-side; it contains platform error text, not user secrets — acceptable for an internal worker log |
| T-03-SC | Tampering | npm installs (youtube-dl-exec, groq-sdk, openai) | high | mitigate | All three are approved in RESEARCH Package Legitimacy Audit; the SUS "too-new" seam flags were manually cleared (long-established orgs) and re-confirmed at the blocking-human checkpoint above before install |
</threat_model>
