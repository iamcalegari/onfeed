<objective>
Prove — rather than rebuild — that an imported recipe is a full catalog citizen: macro-adaptable (RCP-01), a shopping-list contributor (RCP-02), and cook-mode-runnable with timers (RCP-03), all via direct reuse of existing source-agnostic flows (D-11), and verify the SOC-01 (born private) and SOC-05 (credit retention on promotion) claims hold against the shipped code.

Purpose: RCP-01/02/03 are direct reuse per D-11 — adapting an imported recipe creates a generated_pending child (current adaptRecipe behavior, no special-casing), and shopping-list + cook mode work identically because those paths resolve a recipe by id without inspecting source. The risk is not that these need building, but that a latent source assumption breaks them. This plan adds regression tests asserting an imported recipe flows through the adapt/shopping/cook contracts unchanged, verifies SOC-01/SOC-05 hold, and documents the citizenship guarantees in the import module README. If any existing path special-cases source in a way that breaks imports, this plan surfaces it as a gap.
Output: a citizenship regression test suite, verified SOC-01/SOC-05 assertions, and an updated import README documenting full-citizenship reuse.
</objective>
