<objective>
Give imported recipes full search/swipe citizenship (RCP-04): an owner sees their own private imports in their own results with an I/E/T/N match score and a neutral "sua importação" badge, promoted public imports appear in everyone's catalog, and a private import of one user never leaks into another user's (or the public) search.

Purpose: RCP-04 requires imported recipes to appear in search/swipe with an I/E/T/N score identically to any other recipe. D-10 requires the owner to see their private import (owner-scoped) marked with a 🎬 badge, and D-05 means a promoted import keeps source:"imported" + visibility:"public" so it must surface in the public catalog too. The single highest-risk edit in this phase is widening the `hybridSearch` `$vectorSearch.filter` source set to include `"imported"` WITHOUT letting private imports leak into public/other-user queries — D-14's guardrail. This plan makes that edit safely (paired with the visibility guard) and adds the badge on both result surfaces.
Output: the widened + regression-tested `hybridSearch` filter, a search-isolation test (private import of A never in B's results), and the neutral imported badge on ResultCard + DeckCard.
</objective>
