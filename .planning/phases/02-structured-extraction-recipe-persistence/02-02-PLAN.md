<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| transcript/caption → LLM prompt | ASR output + post caption are attacker-influenceable (video creator controls both) and enter the extraction prompt as data |
| LLM output → schema | Model output is validated by zod but its semantic truthfulness (grounding honesty) is not guaranteed by the shape |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-03 | Tampering / Spoofing | transcript/caption in extraction prompt | high | mitigate | Delimited triple-quoted labeled user-turn sections (never in system prompt); explicit system-prompt instruction that transcript/caption are DATA not instructions and instruction-like text inside them is ignored (Pitfall 4). Adversarial-injection fixture exercises this. Defense-in-depth: the confidence gate (Plan 04) + mandatory review (Plan 05) mean a successful injection cannot silently reach public. |
| T-02-04 | Tampering (data integrity) | over-confident grounding | high | mitigate | Enum ordering + explicit grounding criteria + anti-default-to-grounded instruction + (Plan 04) structural noSpeechDetected override so the model's self-graded confidence is never the sole gate. |
| T-02-05 | Information Disclosure | logging of transcript/LLM payloads | medium | mitigate | Do not console.log full LLM request/response or raw transcript (third-party creator content) — reuse recipe.ingestion.ts discipline (CONCERNS.md). Error strings carry stop_reason + a job id, not payload bodies. |
| T-02-SC | Tampering | npm/pip/cargo installs | low | accept | No new packages (RESEARCH Package Legitimacy Audit: zero installs). No [ASSUMED]/[SUS] → no install checkpoint. |
</threat_model>
