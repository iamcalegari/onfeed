<objective>
Close the D-14 security gap so an imported private recipe enters the hybrid I/E/T/N search for its importing user (EXT-04) WITHOUT leaking every user's private imports to everyone. Add an ownerId-aware filter to hybridSearch (private recipes visible only to their creator; non-private visible to all), declare the required filter fields on the Atlas vector index, and add an IDOR-safe optional-userId overload to getRecipeById — all borrowing the established getImportJob single-query owner-filter idiom. Never add 'imported' to the global DEFAULTS.sources without the owner scope in the same change.

Purpose: EXT-04 requires imported recipes to be searchable, but D-14/Pitfall 2 flag that hybridSearch has zero owner-scoping today — naively adding 'imported' to DEFAULTS.sources would broadcast private imports to all users. This plan makes searchability and privacy land together.
Output: owner-scoped hybridSearch + Atlas index filter fields + IDOR-safe getRecipeById overload + a repository test proving cross-user isolation.
</objective>
